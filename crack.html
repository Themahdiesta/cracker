🎯 Hash Cracker

Hashcat & JtR command builder with all attack modes.

Rainbow Tables: Pre-computed hash→plaintext lookup tables. Defeated by salting — adding random bytes before hashing. Even 1-byte salt multiplies rainbow table size by 256x. Most modern systems (Linux $6$, bcrypt, etc.) use salts, making rainbow tables impractical. NTLM hashes are NOT salted, which is why PtH works. Hash Collision: An extremely rare case where two different inputs produce the same hash — theoretically possible but practically negligible for password cracking.
Brute Force Protection: WAFs (Web Application Firewalls) and tools like fail2ban can block dictionary attacks by locking accounts after failed attempts. Always enumerate lockout policies before attacking. Password spraying (one password across many users) avoids per-account lockout.
⚡ 10-Step Systematic Cracking Methodology
ALWAYS FOLLOW THIS ORDER
1
Check potfile first
hashcat -m 1000 hash.txt --show — Hashcat caches cracked hashes in ~/.hashcat/hashcat.potfile. Check FIRST.
2
Online lookup (zero GPU cost)
Paste hash at: hashes.com | crackstation.net | ntlm.pw — NTLM hashes commonly pre-computed online.
3
Straight rockyou (~30% crack rate)
hashcat -m MODE hash.txt /usr/share/wordlists/rockyou.txt — 14.3M passwords. Decompress first: gzip -d rockyou.txt.gz
4
rockyou + best64 rules (~60-70% corporate)
hashcat -m MODE hash.txt rockyou.txt -r /usr/share/hashcat/rules/best64.rule — Handles Password1!, Summer2024! patterns.
5
CeWL target-specific wordlist + rules
cewl https://TARGET -d 3 -m 5 -w target.txt then crack with rules. Company/product/city names = common bases.
6
Policy-filtered wordlist
grep -P '^(?=.*[A-Z])(?=.*[a-z])(?=.*\d).{8,}$' rockyou.txt > policy.txt — Filter to match known policy cuts keyspace 90%+.
7
Known-pattern masks
hashcat -a 3 -m MODE hash.txt '?u?l?l?l?l?l?d?d?s' — Corporate default: Capital+lower+digits+special.
8
PACK masks from cracked passwords
python3 statsgen.py cracked.txt -o pack.hcmask — Cracked passwords reveal exact policy pattern. Install: git clone iphelix/pack
9
Combinator attack
hashcat -a 1 -m MODE hash.txt wordlist.txt wordlist.txt — Catches "winterspring", "pass1234", compound passwords.
10
Incremental (last resort)
hashcat -a 3 -m MODE hash.txt '?a?a?a?a?a?a?a?a' — ⚠️ 8-char full keyspace = years. Only viable for ≤6 chars.

🚨 Common Cracking Errors — Exact Fixes
# Error: "No hashes loaded" → sed -i 's/\r//' hash.txt && sed -i 's/^Database://' hash.txt
Fix: Remove CRLF line endings, BOM chars, or tool prefixes (Database:, *)
🔧 Fix: Inspect with: cat hash.txt | xxd | head — look for 0d0a or efbbbf bytes.
# Error: "Token length exception" → hashcat --identify hash.txt
Fix: Hash truncated or wrong mode. Verify full hash copied. Check mode.
🔧 Fix: Responder NTLMv2 must be one complete unbroken line. Don't truncate.
# Error: "Separator unmatched" → hashcat -m 1000 hash.txt rockyou.txt --username
Fix: File has user:hash format. Add --username flag.
🔧 Fix: secretsdump/NetExec output contains username:hash. Always use --username.
# SSH key fails with hashcat → john --wordlist=rockyou.txt ssh.hash
Fix: Modern OpenSSH aes-256-ctr — hashcat 22921 can't handle it. Use JtR.
🔧 Fix: JtR handles ALL SSH key formats. Hashcat only supports specific older ciphers.
# bcrypt/Argon2/yescrypt extremely slow → john --wordlist=rockyou.txt hash.txt
Fix: Use JtR for $y$ (yescrypt) and Argon2 — hashcat has NO support for these.
🐢 Slow: bcrypt ~105KH/s on RTX3090. For bcrypt try --backend-devices 1 -D 1 (CPU mode).
Methodology: 1) Extract hash → 2) Identify type → 3) Check cracking feasibility → 4) Prepare wordlist + rules → 5) Crack. Hashcat = GPU-optimized, JtR = CPU-friendly.
